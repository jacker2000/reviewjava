1.mySql索引类型：
    1.B-Tree:
        使用平衡树实现索引，是mysql中使用最多的索引类型，
        innodb中，存在两种索引类型：
            1.主键索引
                在索引内容中直接保存数据地址
            2.其他索引
                在索引内容中保存主键索引引用

        在innodb中尽量使用主键索引，速度非常快

    2.Hash：
        把索引值做hash运算，存放到hash表中，使用较少，
            一般是memory引擎使用，因为使用hash表存储，按照常理，hash性能比B-tree效率高很多

           hash索引缺点：
                1.hash索引只能适用于精确的值比较，=,in,或者<>
                2.无法使用索引排序
                3.组合hash索引无法使用部分索引
                4.如果大量索引hash值相同，性能较低

    3.full-text索引：
        全文检索索引，效率低，限制多

    4.R-tree:
        针对空间数据索引，使用很少

2.索引好处
    1.提高表数据的检索效率
    2.如果排序的列是索引列，大大降低排序成本
    3.在分组操作中如果分组条件是索引列，也会提高效率
3.索引缺点：
    索引需要额外的维护成本
