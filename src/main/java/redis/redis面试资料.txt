1.Redis到底是多线程还是单线程？线程安全吗
    总结：
        redis是单线程，线程安全的
        redis可以能够快速执行原因：
            1.绝大部分请求是纯粹内存操作(非常快速)
            2.采用单线程，避免不必要上下文切换和竞争条件
            3.非阻塞IO，IO多路复用
                IO多路复用三种方式：
                    select,poll,epoll,注意：select,poll是线程不安全的，epoll是线程安全的

    redis线程安全问题：
        采用线程封闭观念，把任务封闭在一个线程中，避免线程安全问题
            对于需要多个redis操作复合操作来说，需要锁，而且可能是分布式锁
